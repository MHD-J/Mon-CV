Infrastructure Sécurisée & Proxy Inverse

Nginx | Docker | SSL/TLS | Linux | Bash

📖 Contexte

Projet d'administration système : mise en place d'une infrastructure web sécurisée avec reverse proxy Nginx, conteneurisation Docker et chiffrement SSL.

🏗️ Architecture

Internet → HTTPS → Nginx (Proxy) → Docker Network → [Backend API | DB | Frontend]

⚙️ Fonctionnalités

  • Reverse proxy Nginx avec load balancing
  • Containerisation Docker / Docker Compose
  • Certificats SSL/TLS (Let's Encrypt)
  • Pare-feu automatisé (iptables/UFW)
  • Scripts Bash de déploiement

💻 Configuration Nginx

server {
listen 443 ssl;
ssl_certificate /etc/letsencrypt/live/site/fullchain.pem;
location /api/ { proxy_pass http://backend:5000/; }
location / { proxy_pass http://frontend:80/; }
}
